# Story S2: Harden Supabase Dashboard Configuration

Status: done (code + migration parts; dashboard-only residuals documented)

Severity: P0 — pre-launch hardening

## Story

As an Owner,
I want Supabase Auth and Storage configured per security best practice,
so that we don't ship a known-misconfigured auth flow or a publicly-listable storage bucket.

## Acceptance Criteria

1. Supabase Auth is configured for **PKCE only** — implicit grant flow is disabled
2. Magic-link OTP expiry is set to **10 minutes** (default is 1 hour)
3. Supabase Auth Redirect Allow List contains **only** the exact production and preview URLs (no wildcards, no localhost in production project)
4. `nursery-logos` Storage bucket has **directory listing disabled** — public GET by exact URL still works
5. The implicit-flow branch in `app/auth/callback/page.tsx` is gated behind an opt-in env var (or removed if PKCE migration is complete)
6. CAPTCHA is enabled on the magic-link OTP endpoint (Cloudflare Turnstile or hCaptcha)
