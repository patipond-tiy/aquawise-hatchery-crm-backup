## Why this is split from `01.qa-user-story-audit.md`

The audit doc is a **snapshot** — it should remain readable as a point-in-time
record of where the stories stood on 2026-05-13. Mutating the audit's
Scorecard via the orchestrator step above is allowed because the change is
recorded explicitly (`FIX-AUTO: …` → `DONE-AUTO: …`) — readers can still see
what was originally flagged and how it was resolved.

The `/ultrawork` prompt itself lives here because:

- The prompt may need iteration; co-locating it with the audit lets us track
  prompt-engineering changes without rewriting the audit.
- A future fix pass (e.g., after FIX-REVIEW resolution) can branch from this
  file as `04.fix-review-resolution-ultrawork-command.md` or similar.
