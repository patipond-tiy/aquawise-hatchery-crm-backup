# Story A5: Dashboard Computed Stats and Role-Aware Content

Status: done

## Story

As an Owner (P1 — primary user per persona matrix),
I want the dashboard hero stats to show real numbers from my data,
so that I can trust the three KPIs I check every morning without doubting whether they are placeholders.

## Acceptance Criteria

1. "ล็อตที่กำลังเลี้ยง" chip shows `{active}/{total}` computed from `batches` where status = `active` vs. total batch count
2. "D30 อัตรารอดเฉลี่ย" chip shows the mean D30 across all active batches (or `—` if no data), formatted as `{value}%`
3. "ต้องสั่งใหม่ใน 14 วัน" chip shows the count of customers with `restockIn ≤ 14` and their estimated total PL volume
4. "Continue Watching" cards show the customer's actual current batch reference (not hardcoded `'ล็อต B-2604-A'`)
5. Dashboard is accessible to Owner (★), Manager (✓), Rep (✓). **Lab (—)** should not see the dashboard — redirect to their primary surface (`/batches`). **Auditor** — `auditor` is a planned Phase H3 persona (see `01-personas.md` P5: "planned, Phase H3"); no `auditor` users exist in the H1 population. The H1 gate must include an `auditor` no-op pass-through (allow access without redirect) so the enum is valid from day one, but no real auditor access needs to be tested or verified until H3.
6. All stat values degrade gracefully: if no batches exist, show `0/0` or `—`, not an error
