# Story E2: Auto-Create Alerts from Farm D30 Dips

Status: draft

## Story

As a Manager,
I want to receive an alert when farms report D30 below target on a batch I produced,
so that I can investigate before the pattern spreads.

## Acceptance Criteria

1. A cron/trigger watches `farm_cycle_metrics.d30` rows posted from the AquaWise farm app
2. When ≥2 farms with the same source `batch_id` report D30 < 70%, generate an `alerts` row with `severity = 'medium'` and action text "ตรวจสอบล็อต"
3. When ≥3 farms or any farm reports D30 < 60% on the same `batch_id`, severity escalates to `'high'`
4. Notification delivered per the nursery's Settings → Notifications toggles
5. De-duplication via partial unique index on `(batch_id, alert_kind, week)` — no duplicate alert for the same dip within the same week

### Threshold Logic Summary
```
week = date_trunc('week', now())

dip_count  = COUNT(*) WHERE batch_id = $batch_id AND d30 < 0.70 AND week = $week
severe_dip = EXISTS(   WHERE batch_id = $batch_id AND d30 < 0.60 AND week = $week)

IF dip_count >= 3 OR severe_dip → severity = 'high'
ELSE IF dip_count >= 2          → severity = 'medium'
ELSE                            → no alert
```
