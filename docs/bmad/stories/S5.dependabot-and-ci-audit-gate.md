# Story S5: Dependabot + CI `pnpm audit` Gate

Status: done

Severity: P1 — pre-launch hardening

## Story

As an Owner,
I want automated weekly PRs for dependency upgrades and a CI step that fails builds with high-severity advisories,
so that a `next` (or any other) vulnerability does not sit unpatched while we ship features.

## Acceptance Criteria

1. `.github/dependabot.yml` exists, scans `npm` ecosystem weekly, opens up to 10 PRs at a time, applies a `dependencies` label
2. `.github/workflows/ci.yml` includes a step `pnpm audit --audit-level=high` that **fails the build** on any unaddressed high-severity finding
3. The same workflow also runs `pnpm typecheck`, `pnpm lint`, and `pnpm test`
4. A Dependabot PR landing on a feature branch triggers CI automatically
5. README or `docs/bmad/code-design.md` references the policy for handling Dependabot PRs (review and merge weekly; high-severity must be merged or explicitly waived within 48 hours)

- [x] Task 2 — CI workflow (AC: #2, #3, #4)
  - [ ] Create or update `.github/workflows/ci.yml`:
    ```yaml
    name: CI
    on:
      pull_request:
      push: { branches: [main] }
    jobs:
      gates:
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4
          - uses: pnpm/action-setup@v4
            with: { version: 10 }
          - uses: actions/setup-node@v4
            with: { node-version: 20, cache: pnpm }
          - run: pnpm install --frozen-lockfile
          - run: pnpm audit --audit-level=high
          - run: pnpm typecheck
          - run: pnpm lint
          - run: pnpm test
    ```

## Dev Notes

### Architecture Constraints
- pnpm version is `10.0.0` — use `pnpm/action-setup@v4 with: { version: 10 }`, not the default
- Node version 20+ per `package.json` `engines`
- `pnpm install --frozen-lockfile` enforces lockfile consistency

### Files to Touch

### Edge cases
- If a transitive dependency has a high-severity advisory but no upstream patch exists yet, document in `code-design.md` §19 ("Deferred advisory: X, reason: no patch released; mitigation: Y") and add to `pnpm audit --audit-level=high --ignore=<advisory-id>` so CI doesn't block forever. Re-evaluate weekly.
- Dependabot may open PRs that fail tests on minor breaking changes. The dev team treats those like any other failing PR — fix or close, don't merge red.
