# Story A4: Sign Out

Status: done

## Story

As any authenticated user,
I want to click "ออกจากระบบ" in the left rail and be signed out,
so that I can securely end my session, especially on shared devices.

## Acceptance Criteria

1. Clicking "ออกจากระบบ" in the left rail calls `supabase.auth.signOut()` and clears the session
2. After sign-out, the user is redirected to `/th/login` (or the locale-appropriate login page)
3. After sign-out, navigating to any dashboard route redirects to login (no stale session)
4. TanStack Query cache is cleared on sign-out so the next user does not see stale data
5. Sign-out works in both mock mode and live Supabase mode. In **mock mode** (`USE_MOCK=true`), "works" means: (a) clicking "ออกจากระบบ" triggers the sign-out handler without throwing, (b) the user is redirected to `/th/login`, and (c) `queryClient.clear()` is called (any cached query data is evicted). In **live mode**, "works" additionally means `supabase.auth.signOut()` resolves without error and the Supabase session cookie is cleared.
6. All roles (owner, counter_staff, lab_tech, auditor) can sign out — no role gating
