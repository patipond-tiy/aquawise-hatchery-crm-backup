# Story H4: Quiet Hours Respected at Delivery

Status: draft

## Story

As an Owner,
I want to define quiet hours so no LINE pushes are sent after 9pm or before 7am,
so that automated alerts and broadcasts do not wake up farmers.

## Acceptance Criteria

1. Schema: `notification_settings` gains two nullable TIME columns — `quiet_hours_start` (default 21:00 ICT) and `quiet_hours_end` (default 07:00 ICT)
2. Settings → Notifications grows two time pickers for the hatchery's global quiet-hours window (single window applies to all notification channels)
3. Bot worker checks the recipient hatchery's quiet-hours window before pushing each `line_outbound_events` row; if the current ICT time is inside the quiet window, the event remains `pending` and is re-queued at window-open time (update `scheduled_for` to the next window-open timestamp; do not change status to `failed`)
4. High-severity disease alerts (`payload->>'severity' = 'high'`) bypass quiet hours and are delivered immediately regardless of time; every bypass is logged to `audit_log` with `action='quiet_hours_bypassed'`, `entity_type='line_outbound_events'`, and the event ID
5. Manual rep-initiated `sendLine` from the modal bypasses quiet hours by default — the rep is a human making a deliberate decision; cron-driven events and auto-alert events respect quiet hours
6. Time comparison uses ICT (UTC+7) — store as TIME without timezone in the DB; convert `now()` to ICT before comparing

### Time Zone Handling
Store times as `TIME` (no timezone) in PostgreSQL — they are always interpreted as ICT (UTC+7). In the bot worker (Node.js), convert `Date.now()` to ICT before comparing:

```ts
function nowInICT(): string {
  const d = new Date(Date.now() + 7 * 3600 * 1000)
  return d.toISOString().substring(11, 16) // "HH:MM"
}

function isInQuietHours(now: string, start: string, end: string): boolean {
  // Handle overnight window (e.g., 21:00 – 07:00)
  if (start > end) {
    return now >= start || now < end
  }
  return now >= start && now < end
}
```

### Quiet-Hours Re-Queue Logic
When an event is deferred, calculate the next window-open time and store it as `scheduled_for` (UTC):

```ts
function nextWindowOpenUTC(start: string, end: string): Date {
  // end is the "open" time in ICT; add 7h offset to get UTC
  const [h, m] = end.split(':').map(Number)
  const now = new Date()
  const nextOpen = new Date(now)
  nextOpen.setUTCHours(h - 7, m, 0, 0) // convert ICT to UTC
  if (nextOpen <= now) nextOpen.setUTCDate(nextOpen.getUTCDate() + 1)
  return nextOpen
}
```
