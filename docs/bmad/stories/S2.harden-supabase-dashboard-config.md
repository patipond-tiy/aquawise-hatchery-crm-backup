# Story S2: Harden Supabase Dashboard Configuration

Status: ready-for-dev

Severity: P0 — pre-launch hardening

## Story

As an Owner,
I want Supabase Auth and Storage configured per security best practice,
so that we don't ship a known-misconfigured auth flow or a publicly-listable storage bucket.

## Acceptance Criteria

1. Supabase Auth is configured for **PKCE only** — implicit grant flow is disabled
2. Magic-link OTP expiry is set to **10 minutes** (default is 1 hour)
3. Supabase Auth Redirect Allow List contains **only** the exact production and preview URLs (no wildcards, no localhost in production project)
4. `hatchery-logos` Storage bucket has **directory listing disabled** — public GET by exact URL still works
5. The implicit-flow branch in `app/auth/callback/page.tsx` is gated behind an opt-in env var (or removed if PKCE migration is complete)
6. CAPTCHA is enabled on the magic-link OTP endpoint (Cloudflare Turnstile or hCaptcha)

## Tasks / Subtasks

- [ ] Task 1 — Disable implicit grant flow (AC: #1)
  - [ ] Supabase Dashboard → Auth → Settings → disable "Allow implicit grant flow"
  - [ ] In `app/auth/callback/page.tsx`, gate the implicit-flow branch behind `process.env.ALLOW_IMPLICIT_FLOW === 'true'` — default off
  - [ ] If PKCE is the only path used in production, delete the implicit branch entirely (audit `app/auth/callback/page.tsx` lines 28-48)
- [ ] Task 2 — Shorten OTP expiry (AC: #2)
  - [ ] Supabase Dashboard → Auth → Email → OTP expiry: set to 600 seconds (10 minutes)
- [ ] Task 3 — Lock down redirect allow list (AC: #3)
  - [ ] Supabase Dashboard → Auth → URL Configuration → Redirect Allow List
  - [ ] Remove any wildcard entries; add exact production and preview deploy URLs
  - [ ] Confirm `NEXT_PUBLIC_APP_URL` matches one of the allowed entries
- [ ] Task 4 — Disable Storage directory listing (AC: #4)
  - [ ] Supabase Dashboard → Storage → `hatchery-logos` → Settings → uncheck "Allow listing objects"
  - [ ] Verify via curl: `curl 'https://<project>.supabase.co/storage/v1/object/list/hatchery-logos'` should return 403
  - [ ] Verify public GET still works: `curl 'https://<project>.supabase.co/storage/v1/object/public/hatchery-logos/<uuid>/logo.png'` should return 200 if a logo exists
- [ ] Task 5 — Enable CAPTCHA on OTP (AC: #6)
  - [ ] Supabase Dashboard → Auth → Bot and Abuse Protection → enable Turnstile (free) or hCaptcha
  - [ ] In `app/[locale]/login/page.tsx`, add the CAPTCHA widget and pass `captchaToken` to `signInWithOtp({ options: { captchaToken } })`
  - [ ] Add `NEXT_PUBLIC_TURNSTILE_SITE_KEY` to `.env.example`
- [ ] Task 6 — Smoke test full login flow
  - [ ] Mock-mode: skip — CAPTCHA only fires in live mode
  - [ ] Live mode (preview deploy): request OTP → fill CAPTCHA → click magic link → expect successful sign-in
  - [ ] Attempt to reuse the same magic link in a second browser → expect rejection
