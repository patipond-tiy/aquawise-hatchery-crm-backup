# Story K2: Versioned Batch Read API (`GET /api/v1/batches/:code`)

Status: ready-for-dev

## Story

As **the LINE bot** (acting on behalf of a shrimp farmer claiming a Batch Code),
I want a stable, signed-JWT-authenticated CRM endpoint that returns the full batch payload by `batch_code`,
so that the farmer's pond setup can validate the code and bootstrap PCR + cohort + warning flows (LINE bot Stories K.1, K.7).

## Acceptance Criteria

1. **Route shape:** `GET /api/v1/batches/:code` lives at `app/api/v1/batches/[code]/route.ts`.
2. **Auth:** Every request requires `Authorization: Bearer <ES256-signed JWT>`. The token must carry `iss=line-bot`, `aud=hatchery-crm`, and `exp` Ōēż 15 minutes in the future. Public key for verification is loaded from `CRM_JWT_PUBLIC_KEY` env var (PEM). Invalid signature ŌåÆ 401. Missing token ŌåÆ 401. Expired token ŌåÆ 401 with `WWW-Authenticate: Bearer error="token_expired"`.
3. **Path validation:** `:code` must match `^B-[A-CDEFGHJKMNPQRSTUVWXYZ23456789]{6}$`. Otherwise 400 with `{ error: 'invalid_code_format' }`. (Note the alphabet excludes `0`, `O`, `1`, `I`, `l` to match `mint_batch_code()` in Story K1.)
4. **Happy-path response (200):** body matches the LINE bot's expectation (ADR-018):
   ```jsonc
   {
     "batch_code": "B-A4F2K7",
     "hatchery_id": "h_01HQ...",
     "hatchery_name": "ÓĖ«ÓĖŁÓĖĢÓ╣ĆÓĖŖÓĖŁÓĖŻÓĖĄÓ╣łÓĖ¬ÓĖĪÓĖĖÓĖŚÓĖŻÓĖ¬ÓĖ▓ÓĖäÓĖŻ",
     "hatchery_contact": { "line_oa_id": "@aqx", "phone": "0812345678" } | null,
     "pl_stage": "PL12",
     "source_strain": "Charoen Pokphand SPF",
     "spawn_date": "2026-05-01",
     "pcr_results": { "wssv": "negative", "ehp": "negative", "ihhnv": "negative", "tsv": "negative" },
     "pcr_certificate_url": "https://<signed-url>" | null,
     "valid_until": "2026-06-12T00:00:00+07:00",
     "claimed_by_other": false
   }
   ```
   Fields sourced from `batches`, `pcr_results`, `batch_certs`, and the frozen `hatchery_contact_snapshot` (Story K1).
5. **404 if `batch_code` unknown.** Returns `{ error: 'batch_not_found' }`.
6. **404 if batch exists but `published_at IS NULL`.** Same error body. We do not leak whether a code exists in draft state ŌĆö same 404 either way.
7. **410 if `valid_until <= now()`.** Returns `{ error: 'batch_expired', expired_at: <iso> }`.
8. **409 if the calling LINE user has already claimed a different code AND this code is already claimed by another LINE user.** Body: `{ error: 'claimed_by_other' }`. The endpoint does NOT leak the other claimant's `line_user_id` or display name.
9. **Rate limit:** 10 req/s per `iss` claim. Exceeded ŌåÆ 429 with `Retry-After: 1`. Use an in-memory token bucket (LRU cache keyed on `iss`); acceptable for single-region deployment.
10. **Observability:** every request logs `{ correlation_id (from `X-Correlation-Id` header or fresh UUID), batch_code, status_code, latency_ms }`. Errors capture in Sentry with the correlation id attached.
11. **Token refresh helper:** `POST /api/v1/auth/token` accepts a client-credentials body `{ grant_type: 'client_credentials', client_id, client_secret }`, validates against `LINE_BOT_CLIENT_ID` + `LINE_BOT_CLIENT_SECRET` env vars, returns a 15-min ES256 JWT signed with `CRM_JWT_PRIVATE_KEY`. This is a server-side helper used by the LINE bot at boot ŌĆö not a user-facing endpoint.

## Dev Notes

### Architecture Constraints

- This route lives **outside** `app/[locale]/(dashboard)/...` because it has no locale and no UI. Place under `app/api/v1/` per Next.js App Router conventions; no `BillingGate`, no `next-intl` middleware (the matcher in `proxy.ts` already excludes `/api/*` ŌĆö confirm).
- Service-role Supabase client only. Do not use the cookie-bound client; this endpoint has no user session.
- ES256 (not HS256) because the LINE bot is a separate service that should not hold the CRM's signing secret. Asymmetric keys ŌåÆ CRM signs with private, LINE bot verifies with public, and vice versa for the webhook direction in Story K4.
- No `requireActiveSubscription()` here ŌĆö batch-read is part of the core hatchery-to-farmer trust loop and must not be gated by Stripe state. Locked behind `published_at` (operator opt-in) instead.
- Strict `text/json; charset=utf-8` content type. No HTML error pages.

### API Facade Methods

This story does NOT route through `lib/api/index.ts` ŌĆö the API facade is for *dashboard* data flow. Service-to-service endpoints are independent.

### RBAC

- No `can()` checks. JWT `iss=line-bot` is the only authorisation gate.
- Future: if a second downstream service (e.g., a nursery-side LINE bot) requests this endpoint, extend `iss` whitelist; until then reject any other `iss`.
