## Story

## Dev Notes

- **Current state:** Page does not exist. Only the internal editor view at `/scorecard` exists. (Verified 2026-04-29.)
- **Route shape:** `app/[locale]/h/[slug]/page.tsx` — under the locale-prefixed App Router tree. The `[slug]` matches `hatcheries.slug` (generated on nursery creation in A1).
- **Aggregates:** Must be server-computed (`generateStaticParams` / ISR in F4, or server component fetch). Never pass raw `pcr_results` or `customer_cycles` rows to the client — compute D30 mean, PCR pass rate, and PL volume on the server.
- **RLS consideration:** The existing RLS on `nurseries` and `nursery_brand` assumes authenticated callers. A new policy granting `anon` role `SELECT` on `nursery_brand` (scoped to `public=true` rows) will need careful review — use a DB function with `SECURITY DEFINER` if a view approach exposes too much.
- **ISR:** F4 adds `revalidate: 21600` (6h). This story (F2) should design the page to tolerate ISR from the start so F4 is purely additive.
- **Typography:** Plus Jakarta Sans for headings; Noto Sans Thai for Thai body copy. No dark mode. No customization. Consistent with `components/aw/` token system.
- **Code refs (planned):** `app/[locale]/h/[slug]/page.tsx` (new), new RLS migration (00X), new DB view `scorecard_aggregates_v`, `lib/api/supabase.ts getPublicScorecard` (new), `nursery_brand` table (migration 003 or 006)
