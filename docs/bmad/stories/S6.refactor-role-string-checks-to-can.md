# Story S6: Refactor Role-String Checks to `can()`

Status: ready-for-dev

Severity: P1 — cleanup pre-launch

## Story

As a Developer,
I want every role check in the codebase to go through `can(role, action)` from `lib/rbac.ts`,
so that adding a new role or changing the action matrix is a one-file edit instead of a project-wide grep-and-pray, and the §18(B) anti-pattern stops being half-enforced.

## Acceptance Criteria

1. Every instance of `role === 'owner'` / `role === 'counter_staff'` / `role === 'lab_tech'` / `role === 'auditor'` (and variants with `!==` or `.includes`) in `app/`, `components/`, and `lib/` is replaced with `can(role, '<action>')`
2. Each replacement uses the appropriate action — new actions added to the matrix in `lib/rbac.ts` only when no existing action fits (with PM approval)
3. `lib/rbac.ts` and `architecture.md` §4 stay in sync (matrix matches)
4. A Vitest case asserts each role × each action returns the expected boolean (table-driven test)
5. `pnpm typecheck && pnpm lint && pnpm test` pass
6. Manual click-through of `/th` settings as each role confirms the right UI is hidden/shown (mock mode supports role switching via seed data)
