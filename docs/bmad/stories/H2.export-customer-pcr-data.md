# Story H2: Export Customer / PCR Data

Status: ready-for-dev

## Story

As an Auditor,
I want to download a CSV of customers and a ZIP of PCR reports,
so that I can satisfy compliance requests without manual data gathering.

## Acceptance Criteria

1. "ดาวน์โหลด CSV" (customers) → server action streams a CSV of all `customers` rows scoped to the calling user's hatchery (RLS-enforced); columns include farm name, owner, phone, zone, status, created_at
2. "ดาวน์โหลด ZIP" (PCR history) → server action creates a ZIP of all `pcr-reports/{batch_id}/` blobs for this hatchery, returned as a streamed download
3. "ดาวน์โหลด" (full backup) → server action streams a JSON dump of all entities (customers, batches, pcr_results, alerts) for the hatchery
4. Each export logs a row to `data_exports(id, hatchery_id, kind, requested_by, file_url, completed_at)`
5. Server actions stream data — the entire dataset is never loaded into memory at once
6. Storage blobs are accessed via Supabase signed URLs (not public URLs)
7. `data:export` RBAC action required — auditor role has this permission; counter_staff does not
8. All four export buttons in the Settings → Data tab are wired (currently all no-op)

## Dev Notes

### Architecture Constraints
- All API calls via `lib/api/index.ts` facade — never Supabase directly from pages
- Server actions live next to routes: `app/[locale]/(dashboard)/settings/actions.ts` (or a sub-file `data/actions.ts`)
- `'use client'` opt-in only; server components by default
- No `any` in app code; `as` casts allowed only in adapters (`lib/api/supabase.ts`)
- `lib/stripe/server.ts` pattern: any file that uses the Supabase service-role key must import `'server-only'`

### Streaming Requirement
Do **not** use `getAll()` / collect-then-return patterns. Use a `ReadableStream` (Web Streams API, compatible with Next.js App Router Route Handlers) or pipe via a Node.js `Readable`. For ZIP generation, pipe each blob through the archiver as it arrives — release the previous blob's memory before fetching the next.

### RBAC
- Required action: `data:export`
- Per `lib/rbac.ts`: `auditor` has `data:export`; `owner` has `data:export`; `counter_staff` and `lab_tech` do not
- Pattern: `const role = await getRole(); if (!can(role, 'data:export')) throw new Error('403')`
