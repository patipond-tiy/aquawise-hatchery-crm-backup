## Story

**As** an Owner, **I want to** download the scorecard as a brochure PDF and push it to all my customers via LINE, **so that** they have an offline reference of my nursery's verified quality record.

## Acceptance Criteria

> All AC below are planning-grade hypotheses. Both buttons on the scorecard page are currently toast-only stubs.

- "ดาวน์โหลด PDF" triggers a server-side render of the scorecard and produces a browser-downloadable PDF
  - PDF uses brand-tier typography: Plus Jakarta Sans + Noto Sans Thai; includes nursery logo, key stats (those toggled on in F1), "Verified by AquaWise" stamp, generation timestamp
  - No emoji; no dark mode; voice consistent with `docs/product-spec/07-brand-and-voice.md`

  - Bot worker (G3') renders the scorecard Flex with `nursery_brand` and pushes via @aquawise OA
  - Idempotency: a unique constraint (or partial index) on `(customer_id, template, nursery_id, date_trunc('day', created_at))` prevents duplicate same-day scorecard blasts
- Toast: "ส่งถึง N ฟาร์ม" after submission

## Dev Notes

- **Current state:** Both "ดาวน์โหลด PDF" and "ส่ง LINE" buttons on `scorecard/page.tsx` are toast-only stubs. (Verified 2026-04-29.)
- **PDF library:** Prefer `@react-pdf/renderer` for a React-native render path consistent with C4 (`lib/pdf/cert.tsx`). Puppeteer is an alternative if pixel fidelity is required but adds significant cold-start latency on serverless.
- **LINE dependency:** This story is fully blocked on G3' (the send-only Flex worker). G3' is itself blocked on G1 (LINE bind). Do not begin F3 until G3' is end-to-end verified in production.
- **Cohort selector:** Reuse or extend the multi-recipient pattern from C4's `cert-modal.tsx`. The selector should show only customers with `line_id IS NOT NULL` and display their farm name + LINE bind status.
- **Idempotency:** The daily-blast idempotency constraint prevents the owner from accidentally double-sending the scorecard on the same day. This is distinct from the per-cycle idempotency already in migration 006 — a separate partial index on `line_outbound_events` is needed for this template type.
- **Code refs (planned):** `app/[locale]/(dashboard)/scorecard/page.tsx` (add handlers), `lib/pdf/scorecard.tsx` (new), new server actions in `app/[locale]/(dashboard)/scorecard/actions.ts` (new), `lib/line/templates/scorecard_flex.ts` (new)
