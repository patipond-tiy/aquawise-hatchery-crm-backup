## Dev Notes

- **Current state:** Not implemented. The public scorecard route (`app/[locale]/h/[slug]/page.tsx`) does not exist. F4 is entirely blocked on F2. (Verified 2026-04-29.)
- **ISR approach in App Router:** In Next.js 16 App Router, export `export const revalidate = 21600` at the page module level, or use `generateStaticParams` + `revalidate` segment config. The page will be statically generated at build time for all public hatcheries and revalidated on demand up to once per 6h.
- **OG image:** `app/[locale]/h/[slug]/opengraph-image.tsx` using `ImageResponse` from `next/og`. Keep it simple: nursery logo (if set), display name, D30 mean, PCR pass rate, "Verified by AquaWise" lockup. Size: 1200×630. Font: Plus Jakarta Sans (load via `fetch` from Google Fonts at image-generation time, or bundle the subset).
- **JSON-LD:** Map `Organization` with `name`, `url` (canonical `/{locale}/h/{slug}`), `logo` (from `nursery_brand.logo_url`). For `AggregateRating`, use `ratingValue` = D30 mean (normalized to 0–5 scale or presented as a percentage) and `reviewCount` = batch count. Validate against Google's Rich Results Test before marking .v done.
- **robots.txt:** The existing file (if any) must allow `Allow: /h/`. Check `app/robots.txt` — if it is a static file, convert to `app/robots.ts` (dynamic) if the sitemap is dynamic.
- **Sitemap cadence:** The sitemap query hits `hatcheries JOIN scorecard_settings WHERE public = true`. Use the Supabase server client (service-role, read-only) inside `app/sitemap.ts`. The sitemap itself can carry `lastmod` from `scorecard_settings.updated_at`.
