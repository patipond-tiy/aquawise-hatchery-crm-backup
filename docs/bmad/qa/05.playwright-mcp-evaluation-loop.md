# QA — Playwright-MCP Evaluate-and-Iterate Loop (Nursery CRM)

> **Axis.** `qa/00–04` audit *story↔plan alignment* and *executability*. This
> file is the orthogonal axis: a **functional + visual** loop the QA agent
> runs on the *running app* after the green-build gate, before a story is
> marked `done`. It is per-story, repeatable per release — not a one-shot audit.

## When it runs

In the story lifecycle (`README.md` → "Running a story end-to-end"), **after**
Step 5 (green-build gate: `pnpm typecheck && pnpm lint && pnpm test`) and
**before** the story is marked `done`. Mandatory for any story whose surface
is a `(dashboard)` page, modal, or other rendered UI. Skip only for
pure-backend / migration / config stories (record "no UI surface" in the UAT
file).

## Preconditions (deterministic, no real secrets)

1. `pnpm dev` → app on `http://localhost:3000/th`. Mock mode auto-engages when
   `NEXT_PUBLIC_SUPABASE_URL` is unset (`lib/api` facade → `lib/mock`). **No
   real auth or Supabase needed.** If a story requires an authed dashboard
   state and mock mode does not already short-circuit the login wall, that is a
   **readiness blocker** — see `../READINESS.md` item "no-auth mock entry".
2. Billing/role states are forced via `MOCK_BILLING_STATE` in `.env.local`
   (`trialing-25 | trialing-2 | trial_expired | active | past_due`) — set the
   value the story's AC implies before navigating.
3. Only the **Thai** surface (`/th`) is QA-able; `/en` renders `<ComingSoon />`
   by design.

## The loop

For each story:

1. **Read the contract.** The story's Acceptance Criteria + the matching
   `uat/<epic>.uat.md` scenarios + the design-system contract
   (`../design-system-conformance.md` and the canonical
   `../../../../design-system-v1/preview/{components,colors,type,brand}.html`
   + `ui_kits/crm/`).
2. **Navigate & capture.** `mcp__playwright__browser_navigate` to the route;
   `browser_snapshot` (DOM/a11y tree) + `browser_take_screenshot`.
3. **Drive each UAT scenario.** `browser_click` / `browser_type` /
   `browser_fill_form` / `browser_select_option`; capture a screenshot after
   each meaningful state change.
4. **Evaluate** the render + DOM against:
   - **AC** — every acceptance criterion observably satisfied.
   - **UAT** — each scenario's pass/fail condition.
   - **Design system** — tokens (no raw hex where a `--color-*` token exists),
     **fonts** (Latin body = Inter, headings = Plus Jakarta Sans, Thai = Noto
     Sans Thai, numerics = JetBrains Mono — verify via
     `browser_evaluate` `getComputedStyle`), radii/shadow/spacing feel,
     animation restraint (no opacity-fade entrances), layout vs the
     `design-system-v1` preview/ui-kit baseline.
   - **Brand gates (automatic ❌ if violated):** emoji on a professional
     surface; dark mode; a non-Inter Latin body face; Plus Jakarta Sans used
     as body; banned marketing words in visible copy; English on a `/th`
     surface.
5. **Iterate.** On any defect: file it back to the dev agent (or fix within
   the QA mandate per the existing BMAD fix-loop), then **re-run steps 2–4**.
   Bounded by the same **max-attempt cap** as the BMAD fix-loop; on cap
   exhaustion, record `BLOCKED` with evidence and escalate — do not mark
   `done`.
6. **Record evidence.** Write pass/fail per scenario **inline in the matching
   `uat/<epic>.uat.md`**, each verdict citing the AC#/scenario and a
   screenshot path.

## Evidence convention

- Screenshots → `.qa-artifacts/<story-id>/<step>.png` (gitignored — never
  commit binaries). Reference them by path in the UAT file.
- Every ❌ / ⚠️ must cite a specific AC# or UAT step **and** a screenshot.
  No vibes-based verdicts.

## Hard rules

- **Do not edit story files** from this loop (read-only on `stories/`).
- Deterministic mock data only — no live Supabase, no network-dependent state.
- A green build is a precondition, **not** a substitute — typecheck/lint/test
  passing does not imply the UI is correct.
- Brand-gate violations are automatic ❌, regardless of AC pass.

## Non-goals

- Not a replacement for `pnpm typecheck && pnpm lint && pnpm test`.
- Not authoring automated E2E suites (that is `bmad-qa-generate-e2e-tests`).
- Not pixel-perfect comparison — structural + token + brand-gate evaluation
  against the `design-system-v1` baseline, not a screenshot-diff tool.

## Quick reference

```bash
# 1. deterministic server
pnpm dev                       # http://localhost:3000/th (mock mode)
# 2. (optional) force billing state first
#    .env.local: MOCK_BILLING_STATE=trial_expired
# 3. drive via mcp__playwright__browser_* against the story's route
# 4. font sanity (expected: Noto Sans Thai / Inter chain; .mono → JetBrains Mono;
#    h1 → Plus Jakarta Sans; never Plus Jakarta Sans on body)
```
