## Story

**As** a Farmer (P6), **I want to** open "ข้อความของฉัน" in the @aquawise rich menu and see all my nurseries' threads, **so that** I can chat with my nursery without the conversation cluttering my main LINE chat history.

---

## Acceptance Criteria

> All AC below describe the H3 target. Nothing here should be implemented in H1 or H2.

- The @aquawise LINE rich menu gains an "ข้อความของฉัน" button that opens the LIFF inbox page
- LIFF inbox lists every `chat_threads` row associated with the calling farmer's `line_user_id`, showing nursery name, last message preview, unread count, and relative timestamp
- Tapping a thread opens a 1-on-1 chat view with real-time message delivery via Supabase Realtime on the `chat_messages` table (planned migration 007)
- Nursery side: a new "Inbox" panel in the CRM (new page or right-rail section) shows all active threads with unread badges and last message preview; clicking a thread opens the conversation
- Both sides see a typing indicator and read receipts; read state persisted in `chat_read_receipts` (migration 007)
- Farmer-initiated messages are received by the bot worker and written to `chat_messages`; the CRM surfaces them in the Inbox panel within the Realtime latency budget (≤ 2s on a live connection)
- "เปิดแชท" CTA in existing G3' Flex messages is updated to point to the LIFF inbox (swapping the Phase H1 placeholder LIFF target)

## Dev Notes

- **Relation to G3':** G3' (the send-only worker, in `docs/bmad/stories/`) is the Phase H1 predecessor. G3 is the full two-way inbox that G3' deliberately deferred. Do not conflate them. G3' must be live-verified before G3 planning begins.
- **Cross-repo dependency:** The LIFF pages (inbox, chat thread) and the rich menu change all live in `aquawise-line-bot` (Cloud Run, `asia-southeast1`). This CRM repo contributes only the Inbox panel page and the schema migration. Both repos must share the same Supabase project for Realtime to work — confirm this before starting H3 planning.
- **Schema migration 007:** This migration was referenced but never written during H1/H2. It must define `chat_threads`, `chat_messages`, `chat_read_receipts`. RLS design is non-trivial: the LIFF pages call Supabase with a LINE-issued JWT (not a Supabase magic-link session), so the anon/service-role access pattern needs a custom auth hook or a bot-worker proxy layer.
- **Realtime considerations:** Supabase Realtime on `chat_messages` requires the table to have `REPLICA IDENTITY FULL` set. Message fanout latency budget is ≤ 2s on a live WebSocket connection; this is achievable on the same Supabase project but must be load-tested for a nursery with 30+ active threads.
- **Quiet hours (H4):** Nudge logic must respect `notification_settings.quiet_hours_start/end`. H4 must be live before nudge logic is added to G3.
- **Current state:** Rich menu unchanged; LIFF inbox page does not exist; CRM inbox page does not exist; migration 007 not written. (Verified 2026-04-29.)
- **Code refs (planned — this repo):** `app/[locale]/(dashboard)/inbox/page.tsx` (new), new migration `007_chat_tables.sql`, `lib/api/supabase.ts getThreads / sendMessage` (new)
